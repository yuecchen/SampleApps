Secure ReST API

A simple ReST API that is accessible via {{ apisecure }}
It is secured via basic HTTP authentication. If successful authentication is completed a simple JSON object is returned. If authentication is unsuccessful a 401 message is returned.

Access of Secured API

For now the secure ReST API's is accessible via Curl if they need to be checked against the monitoring solution. A JSON object will be returned back indicating a successful login via valid username and password Example:
curl -i -u admin:secret http://localhost:8080/secure/api
HTTP/1.0 200 OK
Content-Type: application/json
Content-Length: 23
Server: Werkzeug/0.11.4 Python/3.4.3
Date: Sun, 06 Mar 2016 20:05:51 GMT
{
"Secure": "Login"
}
