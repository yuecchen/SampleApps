To Do Task list API

The ToDo API resides here: {{ todoapi }}

A simple task list of To Do items is implemented via a ReST API that is accessible via the /todo/api/tasks URL off of the base URL.
The rest_client program will add, update and delete records repeatedly.
You may stop the rest_client program and click the Reset Task Data button to put things back in a default state. You may also click the Refresh button to observe changes to the data structure.

Consistent Delay API's

The following API's will provide consistent delay times of one thru nine seconds.

  • {{ apione }} [1 second delay]
  • {{ apitwo }} [2 second delay]
  • {{ apithree }} [3 second delay]
  • {{ apifour }} [4 second delay]
  • {{ apifive }} [5 second delay]
  • {{ apisix }} [6 second delay]
  • {{ apiseven }} [7 second delay]
  • {{ apieight }} [8 second delay]
  • {{ apinine }} [9 second delay]

Random Delay API's

The following API's will provide random delay times

  • {{ apisubone }} [random 0 to 1 second delay]
  • {{ apisubnine }} [random 1 to 9 second delay]

Access of Delay API's

For now the delayed ReST API's are accessible via Curl if they need to be checked against the monitoring solution. A JSON object will be returned back indicating the delay time Example:
curl -i http://localhost:8080/delay/api/subnine
HTTP/1.0 200 OK
Content-Type: application/json
Content-Length: 22
Server: Werkzeug/0.11.3 Python/3.4.3i
Date: Thu, 25 Feb 2016 21:00:10 GMTi
{
"Delay:": "3.84"
}
